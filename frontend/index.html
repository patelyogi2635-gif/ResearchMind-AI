What it does

Not a chatbot. An autonomous research agent.

Most AI tools answer from memory. ResearchMind actively searches the live web and GitHub, then synthesizes what it finds into a structured, cited report.

🧠
Query decomposition

Your question is broken into 4 focused sub-questions before any search begins — covering definition, comparison, use cases, and limitations.

🔍
Live web + GitHub search

Tavily MCP searches the live web for articles and docs. GitHub MCP searches repositories for real implementations. Both run for every sub-question.

📊
Source scoring + dedup

All raw results are deduplicated by URL and ranked by relevance. Only the top sources reach the Writer — quality over quantity, always.

📝
Cited report synthesis

The Writer graph synthesizes a markdown report with inline citations, key findings bullets, and a full sources list — in under 2 seconds.

🔗
Model Context Protocol

Tools connect via MCP — an open standard for AI-tool integration. Swap Tavily for any MCP-compatible search provider without touching graph logic.

⚡
Groq inference

All LLM nodes run on Groq's llama-3.3-70b-versatile at ~800 tokens/sec. Planner: 1–2s. Writer: under 2s. Fast, free, and capable.

How it works

Three graphs.
One pipeline.

Each LangGraph graph has exactly one job. State flows between them, accumulating results as it goes. Every node is independently testable.

1
Planner graph

Receives your query. Calls Groq with a JSON prompt to decompose it into 4 sub-questions — covering definition, comparison, use cases, and limitations.

decompose_query node
2
Research graph

Loops over each sub-question. Calls Tavily MCP for web articles, then GitHub MCP for repositories. A scorer deduplicates and ranks all results by relevance.

search_web → search_github → score_sources
3
Writer graph

Selects the top sources, formats them into a numbered prompt, then calls Groq to synthesize a markdown report with inline citations and a sources list.

select_sources → synthesize_report
